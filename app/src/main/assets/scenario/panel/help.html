Help: Panels

In edit mode, panels can be added (button in bottom right), and deleted (menu)

Panel design can be quite complex depending on available space, orientation and shading. In line with the spirit of this app, a simple model is provided to cover the majority of installations.

The panel input has six entries:

EntryDescription
Panel nameA convenient reference for a set of panels (string).
Panel countThe number of panels in this string.
Panel kWpThe maximum that an individual panel will produce. If you have mixed panels, use the lowest value.
OptimisedWhen several strings are connected to the same inverter/MPPT, the simulator needs to know if optimizers are present.

If multiple strings are connected to a single MPPT, and optimised is checked, the maximum of the strings is used.

If multiple strings are connected to a single MPPT, and optimised is not checked, the configuration is assumed to be parallel, and the sum is used. This assumes that the inverter can handle the increased current.

Connected inverter nameThe inverter to which the panels are connected. The simulator needs to understand this in scenarios where multiple inverters are defined.
Connected inverter mpptThe MPPT (Maximum Power Point Tracking) port that the string(s) are attached to.

The lower half of the page shows a 12 month summary of the projected energy generation from this panel string. If it is not present a reminder to fetch is displayed. Usage what-if scenarios that have panels defined but no solar data are not eligible for simulation.

Solar data can be fetched or updated using the button at the bottom of the screen while in edit mode. At the moment only PVGIS is integrated as a source of PV data. In the future additional sources will be added.
